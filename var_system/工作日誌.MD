## v1.0 — 系統建立（2026-04-13）

### 新建
- `var_engine.py`：核心計算引擎（HS / Parametric / MC 三種 VaR、CVaR、Kupiec 回測）
- `var_app.py`：Streamlit 四分頁介面（資料匯入、VaR 結果、圖表分析、回測壓力測試）
- `VaR_系統驗證報告.MD` v1.0：公式推導、數值驗證、Basel III 合規對應（15 節）

### 主要功能
| 功能 | 說明 |
|------|------|
| Historical Simulation VaR | 非參數，取實際損益分位數 |
| Parametric Delta-Normal VaR | 常態假設，Σ=樣本/EWMA |
| Monte Carlo VaR | Cholesky 分解模擬相關路徑 |
| CVaR / Expected Shortfall | 尾端損失條件期望值 |
| Component VaR | 完全可加分解（Parametric 代數精確）|
| Marginal VaR | 數值微分（各方法均支援）|
| EWMA 共變異數 | RiskMetrics λ=0.94 |
| Kupiec POF 回測 | LR 統計量、p-value、Basel 紅綠燈 |
| 壓力測試 | 個別資產衝擊幅度，情境損益計算 |
| Excel 報告下載 | VaR 分解、摘要、報酬序列 |

## v1.2 — 多市場多幣別支援（2026-04-13）

### 需求背景
使用者持有台股（TWD）、美股（USD）、韓股（KRW），需要統一以台幣（TWD）計算 VaR。

### `var_engine.py` 新增
| 項目 | 內容 |
|------|------|
| `DEFAULT_SPOT_FX` | 預設即期匯率常數（TWD=1, USD=32.5, KRW=0.0235, HKD=4.15, JPY=0.22）|
| `detect_asset_currency(ticker)` | 依 Yahoo Finance ticker 後綴自動識別幣別（.TW→TWD, .KS→KRW, 無→USD）|
| `get_market_label(ticker)` | 回傳人類可讀市場標籤（台灣 🇹🇼 / 美國 🇺🇸 / 韓國 🇰🇷 等）|
| `VaRCalculator.__init__` | 新增 `base_currency`、`fx_prices`、`spot_fx` 三個參數 |
| `_build_fx_adjusted_returns()` | 外幣資產回報 = 本地回報 + FX 回報（對數精確分解）|
| `_compute_base_market_values()` | 各資產市值統一乘以即期匯率換算為 TWD |
| `VaRResult` | 新增 `base_currency`、`asset_currencies`、`fx_rates_used`、`local_market_values` |

### `var_app.py` 新增
| 項目 | 內容 |
|------|------|
| 內建多市場範例 | 3 台股（TWD）+ 2 美股（USD）+ 2 韓股（KRW），含模擬 FX 歷史 |
| FX 匯率設定面板 | 即期匯率輸入（USD/KRW/HKD/JPY）+ FX 歷史 CSV 上傳 |
| FX 風險警告 | 未上傳 FX 歷史時自動提示可能低估 |
| 市場別風險貢獻 | Tab 3 新增「市場別 Component VaR 加總」長條圖 |
| 壓力測試 FX 衝擊 | 可個別設定各幣別相對 TWD 升貶幅度 |
| Component VaR 表格 | 同時顯示本地幣別市值與 TWD 等值 |

## v1.5 — ticker 幣別識別修正（2026-04-13）

### 問題
`detect_asset_currency("台積電(2330.TW)")` 回傳 USD，因為字串結尾是「)」而非「.TW」。

### 修改（`var_engine.py`）
- 新增 `import re`
- `detect_asset_currency()` 前置步驟：以 regex `\(([^()]+)\)\s*$` 提取最後括號內的 ticker 代碼
- 修正後：`"台積電(2330.TW)"` → 提取 `"2330.TW"` → 正確識別為 TWD
- `get_market_label()` 連帶修正（呼叫同一函式）
